# Git Workflow

Recommended baseline:

1. create a short-lived feature branch from `main`
2. keep changes scoped to one concern
3. open a pull request early
4. require review before merge
5. tag release-relevant docs updates when architecture, operations, or user-facing behavior changed

Document any project-specific branching, release, or cherry-pick rules here.
