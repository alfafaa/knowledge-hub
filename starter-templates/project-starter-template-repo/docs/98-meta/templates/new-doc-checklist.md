# New Doc Checklist

Before creating a new page:

1. decide the page purpose
2. choose the correct folder
3. use a stable title and filename
4. add the minimal frontmatter
5. decide whether it is local, indexed, or mirrored
6. link to related docs instead of duplicating content
7. add assets under `98-meta/assets/` by default

## Before Publishing A Doc

1. confirm the content is actually useful beyond local drafting
2. set `owner`
3. set `audience`
4. set `visibility`
5. set `publish_targets`
6. verify local links and asset links
7. confirm the content is not stale or speculative

## Quick Questions

Ask these before writing:

- who is the reader?
- what decision or task does this page support?
- is this current state, a proposal, a guide, or a runbook?
- does a page for this already exist?

If you cannot answer those, the page is probably not ready.
