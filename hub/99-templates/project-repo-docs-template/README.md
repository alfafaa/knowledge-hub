# Project Repo Docs Template

This folder is the starter structure developers can copy into a new repository.

Rules:

- create all `mandatory` sections immediately
- create `recommended` sections when the repo needs them
- create `optional` sections only when there is real usage
- use `11-guides/` when the repository owns public, internal, admin, support, or engineering-facing guidance
- use `docs/docs.config.yaml` to control which content syncs to the company hub and which audience receives it
- use `docs/98-meta/assets/` as the default place for repo doc media so main doc folders stay clean
- use page-near asset folders only when one page owns the media tightly
- use `98-meta/` only for shared assets, templates, and workspace support files

This keeps repositories clean without losing scalability.
