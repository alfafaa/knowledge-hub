# Guides

Optional audience-facing guidance layer.

Use this when the repository owns guidance for:

- public users
- internal admins or stakeholders
- support or operations teams
- engineers who consume or contribute to the system

Default publishing intent in `docs.config.yaml`:

- `public/` -> `public-site`
- `internal/` -> `internal-site`
- `engineering/` -> `engineering-site`

Final publication should still depend on approval, owner, status, and review metadata.
